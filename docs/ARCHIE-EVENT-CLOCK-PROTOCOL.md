# Archie Event Clock preregistration

This branch does not implement an Event Clock model. It makes the blocked research candidate executable only as a gate.

`research/event_clock/gate.py` refuses candidate creation unless a verified `archie-linked-recurrence-report/v1` supplies at least three seeds and every seed shows:

- correct carried state improves later-segment bits per byte over reset state;
- transplanted and channel-shuffled state are measurably worse than correct state;
- full and incremental logits remain equivalent within the declared tolerance;
- ordinary fixed-domain retention stays inside the damage bound.

Only after that report clears may the authorization receipt permit a 20–30M parameter prototype. The authorization does not permit a 114M/2B run, a novelty claim, admission, or product integration. The first candidate must retain controls A–E and the attention-only, SSM-only, fixed-rate hierarchy and learned-tokenizer comparisons.
