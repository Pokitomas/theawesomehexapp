# Archie Generation One architecture × quantization campaign

This lane compares model architecture and compression designs without pretending that uploaded source code is a trained model or that reported metrics are admitted evidence.

## Bound source roles

- **Unsloth** is an external training, merge, and export accelerator for the existing Hugging Face Qwen student lane. It is not vendored, trusted as runtime code, or treated as Archie’s brain.
- **Mamba** is a Generation One state-space experiment source. The catalog records Mamba, Mamba-2, and Mamba-3 source identity, but no Archie-trained checkpoint, quantized exporter, or admitted iPhone runtime.
- **RWKV-8** is a design source for matrix-state, low-rank-state, quantized-state, and sparse quantized-state experiments. The cataloged document is not an implementation or checkpoint.
- ArchiveBox, Crawl4AI, and Whisper remain possible archive, web-ingestion, and speech faculties. Kid Pix, PixelWater, hm, and voice-ai are excluded from the neural core.

Exact catalog identities are pinned in `product/archie-architecture-source-catalog.json`. The campaign rejects a modified catalog rather than silently changing its research population.

## Candidate matrix

The immutable campaign contains four Qwen3/GGUF controls:

- Q4_K_M
- Q5_K_M
- Q6_K
- Q8_0 diagnostic baseline

It also contains blocked Generation One cells for:

- Mamba-2 weight-only INT8 and INT4
- Mamba-3 MIMO weight-only INT8 and INT4
- RWKV-8 matrix-state INT8
- RWKV-8 low-rank-state INT8
- RWKV-8 quantized-state INT6
- RWKV-8 sparse quantized-state INT6

These names define experiment cells, not claimed implementations. Only the Qwen/GGUF controls expose a materialization command, and those still require an exact merged checkpoint, converter, quantizer, and training receipt.

## Run the diagnostic campaign

Generate the immutable template:

```text
npm run archie:research:architectures
```

Evaluate a diagnostic result packet:

```text
npm run archie:research:architectures:evaluate -- --results ./architecture-results.json
```

Known missing inputs—checkpoint, exporter, recurrent-state kernel, iPhone runtime, hidden evaluation, reproduction, or physical A15 evidence—are preserved as expected failure receipts so the remaining cells can continue. Unknown failure codes and mismatched comparison bindings remain blockers.

Completed rows may carry digest-bound diagnostic metrics under one training budget, hidden split, grader, and workload set. The report can show a **diagnostic Pareto frontier**, but it always leaves selection ineligible. It rejects self-attested `independent`, `reproduced`, or `physical_a15` booleans. Opaque result digests are references, not proof.

Actual selection belongs only to three separate canonical systems:

1. the six-arm hidden-split intelligence campaign;
2. independently attested physical-iPhone admission;
3. clean-environment reproduction.

## Truth boundary

The campaign is executable research orchestration. It does not validate submitted metrics, provide Mamba or RWKV weights, implement their mobile kernels, prove quality, admit an iPhone model, or select an architecture. Promotion remains human-only after the canonical intelligence, device, and reproduction evidence is real.
