# Frontier surface assembly

The Frontier World Expo constitution and round manifest define hypotheses. This assembly tranche turns one exact round into six runnable prototype fixtures without granting Archie repository write authority or treating headless output as model capability evidence.

## Execution boundary

Archie compiles one AIL program per candidate. The program records the intended norm violation, exact runtime file digests, verification step, and stop-before-publication boundary. Archie never writes.

Maker creates one non-overlapping lease per candidate, writes only that candidate directory, runs the exact offline verifier, preserves the event chain, and issues a receipt. A separate bounded Maker lease writes the assembly and comparison indexes. Merge, deployment, publication, and promotion remain human-controlled.

## Assemble a round

```bash
npm run frontier:expo:derive -- \
  --seed 477 \
  --round issue-477 \
  --output round.json

npm run frontier:surface:assemble -- \
  --round-file round.json \
  --base-sha <exact-40-character-sha> \
  --output-prefix experiments/frontier-world-expo/issue-477
```

Every round produces exactly six roles:

1. familiar control
2. inherited-assumption inversion
3. eccentric transfer from non-software media or ritual
4. recombination of losing candidates
5. low-resource offline form
6. maximal expressive variance

Their runtime grammars are deliberately independent. Candidate code cannot import a shared surface design system, require external assets, or write outside its leased directory.

## Automated capture

```bash
npm run frontier:surface:capture -- \
  --assembly-file experiments/frontier-world-expo/issue-477/assembly.json \
  --output .archie/frontier-surface-captures/issue-477
```

Capture executes every candidate at desktop and phone dimensions, preserves before/after screenshots and interaction traces, blocks non-local network requests, and emits a digest-bound receipt. It is explicitly classified as `headless-browser-fixture`; it cannot satisfy physical-device evidence.

## Redirect without erasing

```bash
npm run frontier:surface:redirect -- \
  --assembly-file experiments/frontier-world-expo/issue-477/assembly.json \
  --candidate <candidate-id> \
  --directive "turn this into a quiet archive of unfinished letters" \
  --base-sha <same-exact-sha>
```

A redirect creates a child candidate with explicit parent lineage and a different artifact digest. The original candidate is not mutated or deleted.

## Claim boundary

These outputs prove deterministic prototype assembly, bounded Maker authority, offline executability, visual-grammar divergence, branch preservation, and automated headless capture. They do not prove frontier-model capability, human preference, real-device behavior, or promotion eligibility. Those claims remain blocked on issues #475, #476, and #478.
